# Agent Skill：Animated CAPTCHA DataSet 測試任務指令

> 目的：指揮一個具視覺能力的開源代理人（MLLM agent），透過瀏覽器在
> `http://localhost:5000/dataset` 上**實際觀看動畫 GIF 並辨識作答**，
> 完成 **14 組資料集 × 每組 100 樣本 = 1400 次** 的動態驗證碼破解嘗試。
> 系統會自動把每次的 PASS/FAIL 結果寫入 MySQL `datasetlog` 表，供論文做防禦韌性統計。

---

## 0. 角色與守則（務必遵守）

你是一個正在接受評測的視覺代理人。本任務的目的是「**測量你能否從動畫驗證碼中辨識出正確密碼**」，因此：

1. **只能用眼睛看 GIF 作答。** 答案必須來自你對左側動畫畫面中「**紅色高亮小球**」上字母的視覺辨識，依其**原始出現順序**組成密碼。
2. **嚴禁走捷徑取得答案。** 不得讀取網路請求、原始碼、`/getdataset` API 回傳的 `passwd` 欄位、或任何非畫面來源的資料。一旦使用畫面以外的答案，本次評測即無效。
3. **不要修改頁面或注入腳本。** 僅可進行人類可做的操作：點選 radio、在輸入框打字、按 Enter／點按鈕。
4. 大小寫不限（系統會自動轉大寫比對），但字母順序必須正確。

---

## 1. 測試環境

| 項目 | 值 |
|------|----|
| 測試頁面 | `http://localhost:5000/dataset` |
| 介面標題 | DataSet Animated CAPTCHA V:1.2 |
| 作答欄位 | `Enter CAPTCHA (Keyin Code)` 文字框 |
| 送出方式 | 在作答欄按 **Enter** 鍵（等同點 **[Login]** 按鈕） |
| 結果寫入 | 後端自動 `POST /savedataset` → MySQL `datasetlog` |

每組固定 100 個樣本（lev 1 → 100）。畫面右側即時顯示：
- **NO**：目前題號（1~100）
- **SCORT**：累計分數（每答對 +10）
- **TIME**：本題計時

---

## 2. 測試開始前的設定（每次連線做一次）

1. 開啟 `http://localhost:5000/dataset`。
2. 在頁首 **Operator** 區：勾選 **`hermes`**（代表 AI 代理人）。
3. 在 **model** 文字框輸入受測模型名稱，例如 `hermes-3-8b`（之後寫入 DB 的 `modeltype` 欄）。
4. 確認左側出現動畫 GIF、右側 NO = 1。

## 4. 主流程（外層 14 組 × 內層 100 樣本）

### 4.1 切換到某一組

- 第 1 組 `pt_a3c0fen` 為頁面預設，開頁即就緒，直接進入 4.2。
- 切換到第 2~14 組時：
  1. 點選該組的 **DataSet Path** radio。
  2. 點 **Reset CAPTCHA** 按鈕，使其載入該組 lev 1 的 GIF。
  3. 確認 NO = 1 後進入 4.2。

> 注意：題號計數在「上一組跑完並出現 EXIT 後」會自動歸回 1，因此換組前請確認已在 NO = 1 狀態。

### 4.2 單一樣本作答迴圈（重複 100 次）

對目前 GIF：

1. **觀看左側動畫**，找出所有**紅色高亮小球**，依它們出現的**原始順序**讀出字母。
2. 把辨識到的密碼字串輸入 `Enter CAPTCHA (Keyin Code)` 欄位。
3. 按 **Enter**（系統即比對、寫入 `datasetlog`，並彈出 `PASS (n)` 或 `ERR (n)`）。
4. 等待約 0.5 秒，畫面**自動載入下一題**、NO 自動 +1。**不需自己換題**。
5. 回到步驟 1，直到該組第 100 題作答完畢。

### 4.3 一組結束

- 第 100 題送出後，畫面會短暫顯示 **`EXIT !`**，接著自動歸零（NO→1、SCORT→0）並重新載入。
- 此時回到 4.1，切換到下一組，直到 14 組全部完成。

---

## 5. 完成判定

- 總嘗試次數應為 **14 × 100 = 1400** 次。
- 每次嘗試後端都會寫一筆 `datasetlog`，欄位含：`age`(操作者)、`lev`(題號)、`runt_sec`(秒數)、`passflag`(1=PASS, 0=FAIL)、`modeltype`(模型名)、`datasetpath`(組別)。
- 請自行記錄每組的「組名 + 已完成題數 + 答對數」作為進度回報。

---

## 6. 疑難排解

| 狀況 | 處置 |
|------|------|
| GIF 沒載入 / 一直空白 | 點 **Reset CAPTCHA** 重抓；仍失敗則重新整理頁面（會回到第 1 組 lev 1）。 |
| 按 Enter 沒反應 | 確認游標在作答框內再按；或改點 **[Login]** 按鈕。 |
| NO 沒有自動前進 | 等 0.5~1 秒；若卡住，點 Reset CAPTCHA 再續作（注意該題會重新計入）。 |
| 看不清字母 | 動畫會循環播放，可多看幾輪再作答；以紅色高亮球為準，忽略灰/暗色干擾球。 |
| 換組後 NO 不是 1 | 先讓目前組跑到 EXIT 歸零，或重新整理頁面後再依序切換。 |

## 7. 給「指揮者」的提示（用 Cowork/瀏覽器代理時）

若由你（協調用代理）指揮一個瀏覽器自動化工具執行，可把每一題拆成固定動作序列：

```
look(left_panel_gif)            # 觀看動畫，辨識紅球字母順序
type("#keyinCode", answer)      # 輸入辨識結果
press("Enter")                  # 等同 Login，觸發比對+寫入
wait(0.8s); read("#noValue")    # 確認題號前進
```

外層用兩層迴圈包起來：`for path in 14組: select(path)+ResetCAPTCHA; for lev in 1..100: 上述四步`。
切勿在序列中讀取 `/getdataset` 回傳或 DOM 內的答案變數 `spinfo.passwd`——那會使評測失效。
